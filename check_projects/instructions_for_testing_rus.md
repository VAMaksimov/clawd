# Инструкция по запуску тестов

Помимо тестов на корректные выходные данные, система автотестирования будет
проверять твою программу и ее исходный код. Чтобы удачно пройти тестирование, необходимо убедится, что твоя программа соответствует приведенным ниже требованиям. Для этого запусти приведенные ниже команды локально и убедись, что они не выводят никаких ошибок.

* **Стилевые тесты.** При написании кода на _C_ и _C++_ необходимо придерживаться _Google C++ Style Guide_. Чтобы проверить, насколько красота твоего кода соответствует стандартам, ты можешь протестировать его с помощью утилиты _clang-format_. В папке ```materials/linters``` лежит файл ```.clang-format```, который содержит необходимые настройки для стилевого теста. Данный конфигурационный файл распространяет свое действие на все файлы, которые лежат с ним в директории или в директориях ниже. Поэтому чтобы данные настройки применились к твоим файлам с исходным кодом, скопируй ```.clang-format``` в папку src. \
  \
  Чтобы запустить проверку на стиль, выполни следующую команду: \
  ```clang-format -n src/sourcefile_name.c``` 

  Необходимая версия clang-format: \
  **Mac** 18.1.8 \
  **Linux** 18.1.8

 * **Статический анализ кода.** Иногда (или не совсем иногда) встречается такое, что корректно компилирующаяся C-программа работает совершенно неправильно или завершается с ошибкой попытки доступа не к своей области памяти. Чтобы предотвратить подобные ошибки на этапе написания программы, используют специальные утилиты, которые анализируют твой исходный код на предмет потенциальных ошибок. Наша система автотестов использует для этого ```cppcheck```.

   Ты можешь протестировать свой исходный код: \
   ```cppcheck --enable=all --suppress=missingIncludeSystem src/soursefile_name.c``` \
   Также можно проверить сразу все файлы с исходным кодом в директории: \
   ```cppcheck --enable=all --suppress=missingIncludeSystem src/```
    
   Необходимая версия cppcheck: \
   **Mac** 2.13 \
   **Linux** 2.13
